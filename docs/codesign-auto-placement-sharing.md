# 自动摆放：给使用者与销售看的说明

> 本文与《Codesign → Render 自动摆放需求》技术文档对应，用**真实使用场景**和**白话**说明「系统能做什么、为什么这样设计、客户会得到什么体验」。  
> 适合：终端用户、售前/销售、项目经理；**不要求**了解渲染器或协议细节。

---

## 一句话说明

在和 AI 对话选好家具、灯具、植物等**真实素材**之后，用户可以**一键（或经简单确认）**在已关联的 Windows 渲染场景里**自动摆好**，并能**预览、撤销、核对**——系统会尽量摆对地方，**不确定时宁可停下来问人**，而不是悄悄摆错。

---

## 客户能感知到的价值

| 维度 | 说明 |
|------|------|
| **省时间** | 大量 SKU 不必手拖手放；搭配方案可快速「落到场景里」看效果。 |
| **可对齐知识库** | 推荐来自项目资料与对话上下文，摆放与「客厅 / 花园」等区域意图一致（在客户确认区域后）。 |
| **可解释、可补救** | 成功 / 失败有原因；歧义时可先预览虚影或进待办，而不是直接提交错误结果。 |
| **和素材平台一致** | 用的都是库里**稳定编号的资产**，便于报价、复用和版本管理。 |

---

## 典型场景举例

### 场景一：别墅项目——「帮我把客厅这套沙发茶几先摆进去」

**情况**：设计师在 Web 端和 AI 聊方案，AI 根据任务书推荐了客厅沙发、茶几、边几，用户点了「在渲染器里摆放」。

**系统怎么做（用户视角）**：

1. 系统知道这些东西是**落地家具**，会按「从上往下找第一个靠谱的水平面」的方式**贴地摆放**（不依赖「材质是不是分得细」——即使模型里桌子和房间被合成一大块面，也能用几何方式落位）。
2. 摆放范围会落在用户已确认的**客厅区域**（例如在场景里框好的「客厅地面」范围），不会默认用「整个场景的盒子」当客厅，避免摆到室外、泳池底、二楼投影等奇怪位置。
3. 若同一根垂线上有多层地面（例如镂空、夹层），系统会标**不确定**，默认**不强行摆**，而是预览或列入待确认，避免沙发落到错误楼层。

**客户话术要点**：「先定房间范围，再自动摆；拿不准会停下来让您看一眼，不会瞎摆。」

---

### 场景二：用户只说「这些全都摆进场景里」

**情况**：用户一次性采纳了几十上百个推荐，但没有说「摆哪一间、哪一片花园」。

**系统怎么做（用户视角）**：

- **不会**用「场景中心」或「摄像机下面」当默认落点去铺满几百件——那样极易摆错区域。
- 会弹出**分区确认**：例如展示当前项目里已有「客厅、主卧、前花园」等区域，或引导用户在渲染器里**画/选范围**、或引用带房间边界的资料；确认后再生成执行计划。
- **室内和室外默认分开**（例如马桶不会自动摆进草坪），除非客户在流程里**明确确认**可以合并处理。

**客户话术要点**：「大批量一定要『先分区再摆』，这是为了防止一半摆进花园、一半摆进楼梯间这种事故。」

---

### 场景三：花园——「沿小路两侧种一排灯 / 沿花园外圈铺一圈植物」

**情况**：用户选了很多户外灯具或植物，希望**沿着路径或边界**布置，而不是在矩形里随机堆。

**系统怎么做（用户视角）**：

1. 优先走渲染器里**路径、散布、笔刷**这类原生能力：用户在场景里**画好路径**（或确认系统根据花园边界生成的预览线），再一键沿路径布置。
2. 路径必须落在已确认的**花园（室外）区域**内，并会避开泳池、采光井等**排除范围**（若项目里配过）。
3. 室外有**坡度限制**：过陡的地方会**失败或明确提示**，而不是把盆栽「压平」在陡坡上（除非将来产品提供显式实验选项）。

**客户话术要点**：「大批量沿边、沿路，用路径/散布更省、更稳；陡坡会告诉您，不会假装没问题。」

---

### 场景四：墙上挂画、吊顶吊灯、桌上的花瓶

**情况**：同一方案里既有落地家具，也有挂画、吊灯、桌面摆件。

**系统怎么做（用户视角）**：

- **挂画、壁灯**：按「贴竖直面」的规则，在有效墙面范围内布置（能力分阶段上线，见产品路线图）。
- **吊灯、风口**：从上方锚定或吊顶逻辑（分阶段）。
- **桌上的花瓶**：适合摆在**已有桌子实例上**或用户点选的面；父子关系清楚时，会先摆大床、再摆床头柜、再摆台灯，**避免小东西先把大床的位置占死**。

**客户话术要点**：「大件优先、搭配跟在大件后面，逻辑接近真实摆场顺序。」

---

### 场景五：执行到一半有冲突或失败

**情况**：自动摆放过程中，有的位置**撞墙、撞家具**，或**坡度不合格**，或**层级歧义**。

**系统怎么做（用户视角）**：

- 可配置策略：例如**继续摆剩下的**、**本批停住**、或在渲染器支持时**整批撤销**。
- 结束会给**汇总**：成功多少、失败多少、主要失败原因分类；必要时带**缩略图或线框预览**，方便对客户展示或内部复盘。

**客户话术要点**：「不是黑箱；有报表、可分批、可和人工步骤衔接。」

## 常见疑问（销售 / 客户）

**问：没有建筑轴网也能用吗？**  
答：可以。规则按**世界坐标、区域范围、几何命中**来设计，不依赖轴网，也不把「材质面」当唯一真理（很多外来模型材质并不干净）。

**问：为什么总要强调「区域 / 分区」？**  
答：这是**防错**核心。没有区域就大批量自动摆，等于让系统在整栋楼里猜「客厅在哪」——猜错的代价远高于多点一次确认。

**问：和「只推荐不买」有什么区别？**  
答：推荐可以停在清单里；**自动摆放**是把**已选定、带真实素材 ID 的方案**在**已关联的渲染场景**里执行，并保留**预览、回滚、审计**空间，适合向业主或内部快速看**真实尺度与占位**。

**问：会不会重复点一次就摆两套？**  
答：设计上支持**幂等**（同一批有稳定批次键），重复执行不应默默翻倍，除非产品显式允许。

## 分阶段能力（便于管理预期）

与客户沟通时，可按「从简到全」说清当前版本范围（与内部 P0–P3 对齐）：

| 阶段 | 用户能听懂的范围 |
|------|------------------|
| **早期** | 单区域 + 地面摆放 + 贴地 + 基础碰撞；不确定时预览或失败说明白。 |
| **随后** | 多房间/多区域目录、常见墙面/吊顶类、父子相对摆放。 |
| **再往后** | 区域内智能撒点、更细的净空与卡口、桌面精准贴合等增强。 |
| **更完整** | 景观铺装/草坪等区域填充、大批量编排优化、地形贴合策略等。 |

具体以实际发版说明为准；本文描述的是**目标体验与原则**，不是单一版本的承诺清单。

## 与内部需求文档的对应关系（可选给技术同事）

- **区域与校验**：对应需求文档中的 Zone Catalog、目标区域指派、下发前校验——产品上表现为「先分区、再执行」。  
- **歧义与批量**：多层地面、大批量确认策略——产品上表现为「保守默认 + 显式放宽需确认」。  
- **路径 / 散布 / 笔刷**：对应工具轨与花园工作流——产品上表现为「沿路、沿边大批量走专用能力」。  
- **室外与坡度**：对应花园 zone、坡度上限与失败码——产品上表现为「陡坡不瞎摆」。
