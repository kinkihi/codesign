# 扩展与第三方集成

与主计划第 5 节对齐：在渲染器与素材平台之外，允许用户按需关联其他产品。

## 1. 集成注册表

| 机制 | 适用场景 |
|------|----------|
| **OAuth2** | SaaS 工具授权拉取/推送文档 |
| **Webhook** | 事件通知（任务完成、版本发布） |
| **MCP** | 极客用户、IDE 内 Agent |
| **Zapier / Make** | 低频、非实时自动化 |

## 2. 开放 API 三类能力

1. **`ingest_url`**：第三方提供只读链接或受控导出，进入 Connector 管道。
2. **`export_brief`**：将当前会话结构化摘要（约束卡片、引用素材 ID）推送到 PM/CAD 插件。
3. **`trigger_action`**：仅暴露 **Action Layer 白名单** 子集；第三方**不得**直接调用渲染器进程。

## 3. 安全边界

- 第三方集成默认 **无 Local Executor 权限**；需用户显式授权与范围（读写/只读）。
- 全量审计：`integration_id`、`principal`、`task_id` 关联。
